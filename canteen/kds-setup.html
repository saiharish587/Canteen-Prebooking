🍳 Kitchen Display System - Setup Guide

Quick Start (5 minutes)

1

Access KDS Interface

Open your browser and go to:

http://localhost/canteen/kds.html
2

Get Admin API Token

You need to login with admin credentials to get a token. Use the API endpoint:

POST http://localhost/canteen-api/api/auth/login

Request body:

{ "email": "admin@bvrit.ac.in", "password": "AdminPassword123" }

Use a tool like Postman or cURL to make this request.

3

Copy the Token

From the login response, copy the access_token value. It will look like:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ...
4

Paste Token in Settings

In the KDS page:

  • Click ⚙️ Settings button
  • Paste the token in the "API Token" field
  • Click "Test Token" to verify it works
  • Click "Save Settings"
5

Start Using KDS

Once orders appear:

  • Pending orders show with yellow border
  • Click "✅ CONFIRM" to accept order
  • Click "👨‍🍳 PREPARING" when you start cooking
  • Click "✨ READY" when order is ready
  • Click "🎉 COMPLETED" when customer collects order

Using cURL to Get Token (Terminal/Command Prompt)

On Windows (PowerShell):

$body = @{ email = "admin@bvrit.ac.in" password = "AdminPassword123" } | ConvertTo-Json $response = Invoke-RestMethod -Uri "http://localhost/canteen-api/api/auth/login" ` -Method POST ` -Headers @{"Content-Type"="application/json"} ` -Body $body $response.data.access_token

On Linux/Mac (Bash):

curl -X POST http://localhost/canteen-api/api/auth/login \ -H "Content-Type: application/json" \ -d '{ "email": "admin@bvrit.ac.in", "password": "AdminPassword123" }'

Troubleshooting

Error: "Disconnected" or Red status badge

The API is not responding. Check:

  • Is the Laravel server running? Check http://localhost/canteen-api/api/health
  • Is Apache/XAMPP running?
  • Check PHP error logs in XAMPP
Error: "Unable to load orders. Check connection and token."

The token might be invalid or permissions are limited. Try:

  • Click Settings (⚙️) and test the token again
  • Make sure you're using an ADMIN account token
  • Login again to get a fresh token
  • Check if there are pending orders in the database
Error: "Connection error: Invalid token format"

The token format is incorrect:

  • Copy the entire access_token value from login response
  • Don't include quotes or extra spaces
  • Token should start with "eyJ"
No orders showing even after token is set

Check these possibilities:

  • Are there any orders in the database with "pending" status?
  • Create a test order first using the API
  • Check if the database connection is working
  • View the debug logs at bottom-left of KDS page
Sound alerts not working

Try these fixes:

  • Enable "Sound Alerts" in Settings (should be checked)
  • Check browser volume is not muted
  • Some browsers block audio - allow audio in browser permissions
  • Try a different browser (Chrome works best)

Features Overview

✅ Real-time Order Display

Orders update automatically every 10 seconds (configurable in settings)

✅ One-Click Status Updates

Advance orders through workflow: Pending → Confirmed → Preparing → Ready → Completed

✅ Sound Alerts

Notification beep when orders arrive, special alert when order is ready

✅ Filter Options

View all orders or filter by status (Pending, Confirmed, Preparing, Ready)

✅ Live Statistics

See count of orders at each stage at a glance

✅ Mobile Responsive

Works on tablets and touch devices for flexible kitchen setup

API Endpoints Used

GET /api/health - Check API status GET /api/auth/login - Login to get token GET /api/admin/orders/pending - Get pending orders PUT /api/admin/orders/{id}/status - Update order status

Environment Info

  • KDS URL: http://localhost/canteen/kds.html
  • API Base: http://localhost/canteen-api/api
  • Database: bvrit_canteen (users, menu_items, orders, etc.)
  • Framework: Laravel (PHP backend)
